# Dashboard 详细设计方案

## 设计理念

**核心目标：让用户在3秒内知道"我现在应该做什么"**

### 设计原则
1. **信息层级清晰** - 最重要的信息最突出
2. **操作一键直达** - 减少点击次数
3. **视觉引导明确** - 用颜色、大小、位置引导用户注意力
4. **反馈即时可见** - 每个操作都有明确反馈

### 关键元素详解

#### 1. 当前学习阶段卡片（最重要）
**位置**：页面最上方，占据最大视觉空间
**内容**：
- 阶段名称（大字体）
- 进度条（视觉化进度）
- 两个大按钮：
  - "继续学习" - 主按钮，accent色
  - "提交作业" - 次要按钮，边框样式

**交互**：
- 点击"继续学习" → 跳转到 `/learning` 页面，自动定位到当前阶段
- 点击"提交作业" → 跳转到 `/learning/submit?stage={当前阶段ID}`

#### 2. 学习材料卡片（左侧）
**培训会议室信息**：
- 🏠 图标突出显示
- 房间名和密码
- "显示密码"按钮（默认隐藏）
- 背景色区分（浅蓝色或浅金色）

**其他材料**：
- 视频教程（数量）
- 学习文档（数量）
- 练习题（数量）
- 每项可点击展开详情

#### 3. 待办事项卡片（右侧）
**显示内容**：
- 需要提交的作业（带截止日期）
- 需要查看的反馈
- 需要完成的练习

**视觉设计**：
- 紧急事项用红色/橙色标记
- 已完成的事项显示删除线
- 可点击直接跳转

#### 4. 统计和通知（下方）
**我的统计**：
- 学习天数
- 完成作业数
- 排名（可选，激励作用）

**最新通知**：
- 作业审核结果
- 新材料发布
- 系统通知
- 最多显示3-5条

#### 5. 学习路径（底部）
**可视化进度**：
- 显示所有学习阶段
- 用图标表示状态：
  - ✅ 已完成
  - 🔄 进行中
  - ⭕ 未开始
- 点击可查看各阶段详情

## 四、技术实现要点

### 1. 数据获取优化
```typescript
// 使用React Query或SWR进行数据缓存
const { data, isLoading, error } = useSWR('/api/dashboard', fetcher, {
  refreshInterval: 30000, // 30秒自动刷新
  revalidateOnFocus: true, // 窗口获得焦点时重新验证
});
```

### 2. 加载状态
- 骨架屏（Skeleton）而不是简单的"加载中..."
- 保持页面结构，只显示占位符

### 3. 错误处理
- 友好的错误提示
- 提供重试按钮
- 记录错误日志

### 4. 响应式设计
- 移动端：卡片垂直堆叠
- 平板：2列布局
- 桌面：3-4列布局

### 5. 性能优化
- 懒加载非关键内容
- 图片优化
- 代码分割

## 五、用户体验指标

### 成功标准
1. **首次内容绘制（FCP）** < 1.5秒
2. **最大内容绘制（LCP）** < 2.5秒
3. **首次交互时间（TTI）** < 3.5秒
4. **用户从登录到开始学习** < 30秒
5. **Dashboard跳出率** < 20%

### A/B测试建议
- 测试不同的CTA按钮文案
- 测试不同的卡片布局
- 测试不同的颜色方案

## 七、设计规范

### 颜色系统
- **主色（Accent）**：金色 #D4AF37 - 用于主要按钮和强调
- **背景色**：深灰 #0A0A0A - 主背景
- **卡片背景**：中灰 #171717 - 卡片背景
- **文字色**：
  - 主要文字：#FFFFFF
  - 次要文字：#A3A3A3
  - 禁用文字：#525252

### 间距系统
- 小间距：8px
- 中间距：16px
- 大间距：24px
- 超大间距：32px

### 字体大小
- 标题1：36px
- 标题2：24px
- 标题3：20px
- 正文：16px
- 小字：14px

这个方案的核心是**让用户一眼就知道下一步该做什么**，通过视觉层级、颜色对比、大小差异来引导用户的注意力。
